# Tree View for Family Show

## What We're Building

An alternate "Tree View" for the family show page alongside the existing Graph View. The tree view renders the same `PersonGraph` data as an indented outline organized by generation — ancestors at the top-left cascading down-right to descendants. Person cards are compact minimal rows (photo + name left, dates right, gender accent via left border). Users switch between views via a segmented pill toggle in the toolbar.

## Why This Approach

The graph view (CSS Grid + SVG connectors) is powerful but visually dense. The tree view provides a cleaner, more scannable alternative that emphasizes generational structure through indentation and vertical stacking rather than spatial layout and connector lines. The indented outline reads naturally like a document and handles varying tree sizes gracefully.

## Key Decisions

- **Person cards — Minimal Rows**: Single-line cards with 32px photo circle + name on left, birth–death dates on right. Gender via `border-l-2` accent (blue-400/pink-400). Focus state: dark background (`bg-ds-primary-container` / `text-ds-on-primary`). Duplicated: `opacity-50 border-dashed`.
- **Layout — Indented Outline**: Each generation indents further right (incremental `pl-` steps). Cards stack vertically within each generation group. Generation labels (`text-[10px] uppercase tracking-wider`) above each group.
- **Toolbar — Segmented Pill**: Two icons (`hero-share` for graph, `hero-queue-list` for tree) in a pill container (`bg-ds-surface-highest rounded-sm p-0.5`). Active view: `bg-ds-on-surface text-ds-surface`. Separated from action icons by a `1px` divider.
- **View mode in URL**: `?view=tree` or `?view=graph` (default). Persists across navigation via `handle_params`.
- **Same interaction model**: First tap focuses person (push_patch), second tap navigates to person show. Same `focus_person` event handler. Same tree depth controls apply.
- **Same data source**: Both views consume the existing `PersonGraph` struct. The tree view groups `graph.nodes` by `row` and renders each row-group as a generation section with indentation proportional to the row index.

## Implementation

### 1. Add `view_mode` assign to `FamilyLive.Show`

- Add `:view_mode` assign in `mount/3` (default `"graph"`)
- Parse `params["view"]` in `handle_params/3` — `"tree"` or `"graph"` (default)
- Add `handle_event("switch_view", ...)` that does `push_patch` with `view` param
- Update `family_path/3` helper to include `view` param when not default

### 2. Add segmented pill toggle in the toolbar

- Desktop: segmented pill in the right-side actions area, before the kinship icon, separated by a `1px` divider
- Mobile nav drawer: add a "Tree View" / "Graph View" nav action that switches mode

### 3. Create `TreeComponent` in `lib/web/live/family_live/tree_component.ex`

- `tree_canvas/1` function component receiving `graph`, `family_id`, `organization`
- Groups `graph.nodes` (filtering `:person` type only) by `row`
- Renders each row-group as a generation section with:
  - Generation label (computed from row offset relative to focus person's row)
  - Indentation via `pl-` classes proportional to row index
  - Vertical stack of minimal-row person cards
- `tree_person_row/1` function component for each person card:
  - Photo circle (32px), name, dates, gender border accent
  - Focus state (dark background)
  - Duplicated state (opacity + dashed border)
  - `phx-click="focus_person"` with `phx-value-id`

### 4. Wire up the template

- In `show.html.heex`, conditionally render `graph_canvas` or `tree_canvas` based on `@view_mode`
- Both wrapped in the same `#tree-canvas` container div
