# E2E Testing Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Set up `phoenix_test_playwright` for browser-based e2e tests and add a gallery navigation test covering the gallery list → gallery show → lightbox flow.

**Architecture:** `Web.E2ECase` wraps `PhoenixTest.Playwright.Case` with `async: true` and `@moduletag :e2e`. The Playwright supervisor starts in `test_helper.exs` and drives a real Chromium browser against the running Phoenix endpoint. The Ecto sandbox is shared with the browser via a User-Agent header, so test DB data is visible to the browser session.

**Tech Stack:** `phoenix_test_playwright ~> 0.13`, Playwright (Chromium), Ecto SQL Sandbox, ExUnit

defmodule Web.E2ECase do
  use ExUnit.CaseTemplate

  using do
    quote do
      use PhoenixTest.Playwright.Case, async: true
      @moduletag :e2e
      import Web.E2ECase
    end
  end

  def wait_liveview(conn) do
    PhoenixTest.assert_has(conn, "body .phx-connected")
  end

### Task 6: Write and run the gallery navigation e2e test

**Files:**
- Create: `test/web/e2e/gallery_navigation_test.exs`

**Step 1: Write the test**

```elixir
defmodule Web.E2E.GalleryNavigationTest do
  use Web.E2ECase

  alias Family.Galleries
  alias Family.Galleries.Photo
  alias Family.Repo

  setup do
    {:ok, gallery} = Galleries.create_gallery(%{name: "Test Gallery"})

    # Insert a processed photo directly — bypasses Oban job since no real
    # image processing is needed; status: "processed" makes it render as a
    # clickable img card rather than a pending placeholder.
    {:ok, photo} =
      %Photo{}
      |> Photo.changeset(%{
        gallery_id: gallery.id,
        original_path: "test/fixtures/test_image.jpg",
        original_filename: "test.jpg",
        content_type: "image/jpeg",
        status: "processed"
      })
      |> Repo.insert()

    %{gallery: gallery, photo: photo}
  end

  test "navigate from gallery list to a gallery and open a photo", %{conn: conn, gallery: gallery} do
    conn
    |> visit(~p"/galleries")
    |> wait_liveview()
    |> click_link(gallery.name)
    |> wait_liveview()
    |> click("#photo-grid [id^='photos-']")
    |> assert_has("#lightbox")
  end
end
```

**Step 2: Run just this test**

```bash
mix test test/web/e2e/gallery_navigation_test.exs --trace
```

Expected: 1 test passes. Chromium launches, navigates to `/galleries`, clicks the gallery card, clicks the photo, and the `#lightbox` element appears.

If the test fails, check:
- `body .phx-connected` selector: LiveView adds `phx-connected` class to `<body>` once the WebSocket connects. If this times out, the server may not be reachable — confirm `server: true` is set.
- `[id^='photos-']` selector: photo stream items get ids like `photos-{photo.id}`. If no photos are visible, the sandbox checkout may not be shared with the browser process. Verify the Ecto sandbox advanced setup is followed (the `PhoenixTest.Playwright.Case` handles this via User-Agent metadata automatically when `async: true`).

**Step 3: Run full test suite**

```bash
mix test
```

Expected: all tests pass including the new e2e test.
